## 3. Protocol Wars

### 3.1 MCP (Model Context Protocol)

- **Creator**: Anthropic (November 2024), donated to Linux Foundation December 2025
- **Adoption**: 97M+ monthly SDK downloads, 10,000+ public servers, 6,400+ in official registry
- **Adopters**: OpenAI, Google, Microsoft, Block, and thousands of developers
- **Purpose**: Connects AI agents to external tools and data sources
- **Challenge**: Consuming 40-50% of context windows before agents do work (March 2026).
  Perplexity CTO announced move away from MCP toward traditional APIs.

### 3.2 A2A (Agent-to-Agent Protocol)

- **Creator**: Google, donated to Linux Foundation
- **Purpose**: Agent-to-agent discovery, collaboration, and task delegation
- **Relationship to MCP**: Complementary. MCP = agent-to-tool. A2A = agent-to-agent.
- **Adopters**: Google ADK (native), CrewAI (added), Gemini CLI (PR merged)
- **Status**: Becoming industry standard alongside MCP

### 3.3 dharma_swarm Protocol Position

dharma_swarm currently uses neither MCP nor A2A natively. It uses:
- **Provider abstraction** (`providers.py`): 9 LLM providers via direct API
- **Stigmergy** (`StigmergyStore`): Indirect agent-to-agent coordination via pheromone marks
- **Message bus** (`message_bus.py`): Async SQLite pub/sub

**Strategic implication**: Implementing A2A would give dharma_swarm interoperability with
the entire Google/ADK ecosystem. MCP is already available via Claude Code subprocess. The
question is whether to adopt these protocols natively or continue with the stigmergy model
(which has properties neither protocol offers -- evolutionary persistence, fitness-weighted
routing, contemplative gating).

[MCP adoption](https://www.pento.ai/blog/a-year-of-mcp-2025-review),

## 4. Welfare-Ton Framework

### 4.1 The Formula

From `~/jagat_kalyan/WELFARE_TONS_SPEC.md` (v2.0):

$$W = C \\times E \\times A \\times B \\times V \\times P$$

| Factor | Name | Domain | What It Measures |
|--------|------|--------|-----------------|
| **C** | Verified CO2 tons | [0, inf) | Net tCO2e sequestered/avoided per year |
| **E** | Employment factor | [0, 5.0] | Job creation density x quality |
| **A** | Community agency | [0, 1.0] | Consent, ownership, self-determination |
| **B** | Biodiversity co-benefit | [0.8, 1.5] | Restoration type quality |
| **V** | Verification confidence | [0, 1.0] | Measurement trust level |
| **P** | Permanence factor | [0, 1.0] | Risk-adjusted durability |

**Critical design properties**:
- **Multiplicative**: Zero in ANY factor kills the entire product. No trading ecological
  harm for social benefit. No greenwashing.
- **Conservative defaults**: Missing data defaults to floor values. Rewards transparency.
- **Compatible with standards**: Carbon follows Verra VCS / Gold Standard methodology.
  Employment references ILO Decent Work. Biodiversity references Kunming-Montreal GBF.
  Verification compatible with ICVCM Core Carbon Principles.
- **Auditable**: Every welfare-ton decomposes into 6 factors, each into sub-indicators,
  each with cited source data.
