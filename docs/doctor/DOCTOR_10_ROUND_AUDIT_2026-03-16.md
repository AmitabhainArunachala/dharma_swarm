## Stable Findings After 10 Rounds

### High-confidence defects

1. Frontend/backend route contract drift
The new typed dashboard API helpers call routes that do not exist in the backend:
- `fetchSwarmOverview()` calls `/api/swarm/overview`, but the backend exposes `/api/overview`: [api.ts](/Users/dhyana/dharma_swarm/dashboard/src/lib/api.ts#L103), [health.py](/Users/dhyana/dharma_swarm/api/routers/health.py#L79)
- task routes call `/api/tasks*`, but the backend exposes `/api/commands/task` and `/api/commands/tasks`: [api.ts](/Users/dhyana/dharma_swarm/dashboard/src/lib/api.ts#L119), [commands.py](/Users/dhyana/dharma_swarm/api/routers/commands.py#L27), [commands.py](/Users/dhyana/dharma_swarm/api/routers/commands.py#L54)
- traces call `/api/traces`, but the backend exposes `/api/commands/traces`: [api.ts](/Users/dhyana/dharma_swarm/dashboard/src/lib/api.ts#L177), [commands.py](/Users/dhyana/dharma_swarm/api/routers/commands.py#L77)
- ontology calls `/api/ontology`, but the backend exposes `/api/ontology/types` and related endpoints: [api.ts](/Users/dhyana/dharma_swarm/dashboard/src/lib/api.ts#L198), [ontology.py](/Users/dhyana/dharma_swarm/api/routers/ontology.py#L43)
- stigmergy calls `/api/stigmergy`, but the backend exposes `/api/stigmergy/marks`, `/hot-paths`, `/high-salience`, `/heatmap`, and related endpoints: [api.ts](/Users/dhyana/dharma_swarm/dashboard/src/lib/api.ts#L204), [stigmergy.py](/Users/dhyana/dharma_swarm/api/routers/stigmergy.py#L32)

2. Provider label drift
`CONDUCTOR_CODEX_CONFIG` is declared as `ProviderType.CODEX` but uses a Claude model string, and `PersistentAgent` routes `ProviderType.CODEX` to `"anthropic"`:
- [conductors.py](/Users/dhyana/dharma_swarm/dharma_swarm/conductors.py#L72)
- [persistent_agent.py](/Users/dhyana/dharma_swarm/dharma_swarm/persistent_agent.py#L26)

3. Lifecycle gap
The current telic loop still does not explicitly represent `ExecutionLease`, `RoutingBias`, or `ProjectionRefresh`. The highest-risk missing step is `ExecutionLease` because it is where idempotent retry identity normally lives.

4. `state_dir` isolation leak
`build_agent_context(state_dir=...)` advertises an override but still pulls shared notes from global directories through `read_agent_notes()`: [context.py](/Users/dhyana/dharma_swarm/dharma_swarm/context.py#L806), [context.py](/Users/dhyana/dharma_swarm/dharma_swarm/context.py#L1181)

5. Typed fetch wrapper contract drift
`_fetchWrapped()` treats the backend body as `T`, but the backend actually returns `ApiResponse(data=...)`. The legacy `apiFetch()` unwraps correctly; the new typed helper path does not: [api.ts](/Users/dhyana/dharma_swarm/dashboard/src/lib/api.ts#L35), [api.ts](/Users/dhyana/dharma_swarm/dashboard/src/lib/api.ts#L278)
