# WISDOM LAYER: Self-Evolving Organism Remaining Build

## 1. The Strategic Truth

The hard part is no longer “can the system produce research?” That is already true.

The hard part is now control:

- can it change itself without lying to itself
- can it represent new structures without fragmenting runtime truth
- can it generate new work without inventing a second scheduler

That is why Phase 5 is the real hinge. If optimization is sloppy, the later layers become noise amplifiers.

## 2. What Matters Most

### First Priority: Truthful Optimization

The optimizer bridge must be boring in the right way:

- runtime-field first
- archive-aware
- rollback-safe
- reward-driven

If it is elegant but bypasses those constraints, it is wrong.

### Second Priority: Topology as Data, Not Special Cases

The topology genome should not be a sprawling mini-framework. It should be a compact contract that compiles into the workflow system already present.

### Third Priority: Curriculum Without Empire-Building

The curriculum engine must generate frontier tasks, not a new civilization of task-management abstractions.

## 3. Honest Cut Lines

If time pressure hits, cut depth in this order:

1. Phase 9 breadth
2. Phase 8 target count
3. Phase 7 sophistication of heuristics
4. Phase 6 advanced condition semantics

Do not cut:

- rollback semantics in Phase 5
- provenance semantics in Phase 6
- one-runtime discipline in Phase 7

## 4. External Tool Reality

Concrete tools worth supporting, but only behind guards:

- `nevergrad` for black-box runtime optimization
- `textgrad` or equivalent prompt-gradient tooling for prompt-only experiments

Concrete tools not to center in the live path:

- PPO / DPO / GRPO / VERL / LoRA execution frameworks
- educational autodiff systems

## 5. Telos Check

This build remains aligned when:

- truthfulness improves faster than cleverness
- provenance becomes denser, not weaker
- mutations are explainable after the fact
- new tasks arise from evidence, not restlessness

It drifts when:

- optimization chases score without grounding
- topology becomes harder to understand than the work itself
- curriculum generates volume instead of difficulty

## 6. Recommendation to the Next Agent

Treat the current state as a restrained success:

- Phases 0-4 gave the organism eyes and a scoring function
- Phases 5-9 must give it adaptation without delusion

The shortest faithful path is still:

1. optimizer bridge
2. topology genome
3. curriculum engine
4. export/install split
5. offline export stub

No shortcuts through “smart” rewrites.
